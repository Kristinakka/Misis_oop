# Лабораторная работа №3: Наследование и полиморфизм

## 1. Цель работы
Изучение механизмов наследования и полиморфизма в ООП на примере иерархии классов оружия.

## 2. Описание реализованной иерархии классов

### Базовый класс `Weapon` 
Хранит общие характеристики оружия и содержит методы:
- `attack()` — атака с уменьшением прочности
- `repair()` — восстановление прочности
- `__str__()`, `__repr__()`, `__eq__()`

### Дочерний класс `Firearms` (огнестрельное оружие)
**Новые атрибуты:**
- `_caliber` — калибр (мм)
- `_magazine_capacity` — емкость магазина

**Новый метод:** `reload()` — перезарядка

**Переопределенный метод:** `__str__()`

**Добавленные методы для полиморфизма:** `calculate_price()`, `display_info()`

### Дочерний класс `Edged` (холодное оружие)
**Новые атрибуты:**
- `_blade_length` — длина клинка (см)
- `_material` — материал

**Новый метод:** `sharpen()` — заточка

**Переопределенный метод:** `__str__()`

**Добавленные методы для полиморфизма:** `calculate_price()`, `display_info()`

### Класс `WeaponCollection` (коллекция из ЛР-2)
- Хранение объектов разных типов
- Фильтрация по типам через `isinstance()`
- Полиморфные методы

## 3. Демонстрация работы

### Сценарии:
1. Создание объектов. Базовый + 2 дочерних типа 
2. Новые методы. `reload()` и `sharpen()` 
3. Методы базового класса. `attack()`, `repair()`, `status` 
4. Полиморфизм.`calculate_price()` для разных типов
5. Коллекция. Хранение смешанных объектов
6. Филбтрация `isinstance()` для отбора по типу
7. Полиморфизм без условий.`display_info()` для всех объектов.
8. Агрегация. Суммарная стоимость коллекции
9. Проверка типов.Демонстрация `isinstance()` 

## 4. Вывод

В ходе работы изучены:

- **Наследование** — создание дочерних классов с добавлением новых атрибутов и методов, использование `super()`
- **Полиморфизм** — одинаковые методы (`calculate_price()`, `display_info()`) дают разный результат для разных типов
- **Инкапсуляция** — использование защищенных атрибутов
- **Коллекции** — хранение объектов разных типов с фильтрацией
- **Проверка типов** — использование `isinstance()`
