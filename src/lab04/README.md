## 2. Описание интерфейсов

В лабораторной работе создано 3 интерфейса (абстрактных базовых класса).

### 2.1 Printable — интерфейс для вывода информации
class Printable(ABC):
    @abstractmethod
    def get_info(self) -> str:
        """Вернуть строку с краткой информацией об оружии"""
        pass
Назначение: Обязывает классы реализовать метод get_info(), который возвращает строковое представление объекта.

2.2 Repairable — интерфейс для ремонта
class Repairable(ABC):
    @abstractmethod
    def repair_weapon(self) -> str:
        """Восстановить прочность оружия"""
        pass
Назначение: Обязывает классы реализовать метод repair_weapon(), который восстанавливает прочность оружия.

2.3 Comparable — интерфейс для сравнения
class Comparable(ABC):
    @abstractmethod
    def compare_to(self, other) -> int:
        """Сравнить текущий объект с другим
        Возвращает:
        -1 если текущий меньше
        0 если равны
        1 если текущий больше
        """
        pass
Назначение: Обязывает классы реализовать метод compare_to(), который позволяет сравнивать объекты по урону.

7. Вывод
В ходе выполнения лабораторной работы были изучены и применены на практике следующие концепции:

7.1 Абстрактные базовые классы (ABC)
Созданы абстрактные классы с помощью ABC и @abstractmethod

Абстрактные методы не имеют реализации в интерфейсе

7.2 Интерфейсы как контракты поведения
Интерфейсы определяют, что должен уметь класс, но не определяют как

Классы, реализующие интерфейс, обязаны предоставить реализацию всех абстрактных методов

7.3 Полиморфизм через интерфейсы
Единый интерфейс Printable позволяет работать с разными типами объектов (огнестрельное и холодное оружие)

Вызов get_info() приводит к разному поведению в разных классах

7.4 Проверка типов через isinstance()
Позволяет проверить, реализует ли объект определённый интерфейс

Используется для фильтрации коллекций

7.5 Множественная реализация интерфейсов
Классы могут реализовывать несколько интерфейсов одновременно

class Firearms(Weapon, Printable, Repairable, Comparable)

7.6 Интерфейс как тип данных
Функции могут принимать параметры типа list[Printable]


Вывод: Интерфейсы позволяют создавать гибкую, расширяемую архитектуру, где классы связаны контрактами поведения, а не конкретными реализациями. Это упрощает тестирование, замену компонентов и добавление новых классов без изменения существующего
