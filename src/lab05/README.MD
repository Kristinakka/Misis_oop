## 2. Реализованные функции и стратегии

### 2.1 Функции-стратегии для сортировки

| Функция | Описание |
|---------|----------|
| `by_name(weapon)` | Сортировка по имени оружия |
| `by_damage(weapon)` | Сортировка по урону |
| `by_rarity(weapon)` | Сортировка по редкости (легендарное → редкое → обычное) |
| `by_damage_then_name(weapon)` | Сортировка сначала по урону, затем по имени |
| `by_price(weapon)` | Сортировка по цене |

### 2.2 Функции-фильтры

| Функция | Описание |
|---------|----------|
| `is_legendary(weapon)` | Отбирает только легендарное оружие |
| `is_high_damage(weapon, min_damage=100)` | Отбирает оружие с уроном выше порога |
| `is_repairable(weapon)` | Отбирает оружие, нуждающееся в ремонте |
| `is_firearms(weapon)` | Отбирает только огнестрельное оружие |
| `is_edged(weapon)` | Отбирает только холодное оружие |

### 2.3 Фабрики функций

| Фабрика | Описание |
|---------|----------|
| `make_damage_filter(min_damage, max_damage=None)` | Создаёт фильтр по диапазону урона |
| `make_rarity_filter(rare_level)` | Создаёт фильтр по редкости |
| `make_price_multiplier(multiplier)` | Создаёт функцию для изменения цены |
| `apply_discount(percent)` | Создаёт функцию для применения скидки |

### 2.4 Функции для `map()`

| Функция | Описание |
|---------|----------|
| `to_string(weapon)` | Преобразует оружие в строку |
| `to_info_dict(weapon)` | Преобразует оружие в словарь |
| `apply_discount(percent)` | Применяет скидку к цене оружия |

### 2.5 Callable-объекты (паттерн «Стратегия»)

| Класс | Описание |
|-------|----------|
| `DamageUpgradeStrategy` | Увеличивает урон оружия на заданное значение |
| `RarityUpgradeStrategy` | Повышает редкость оружия (обычное → редкое → легендарное) |
| `RepairStrategy` | Восстанавливает прочность оружия до 100 |

### 2.6 Методы коллекции `Inventory`

| Метод | Описание |
|-------|----------|
| `sort_by(key_func, reverse=False)` | Сортировка коллекции с использованием переданной функции-ключа |
| `filter_by(predicate)` | Фильтрация коллекции с использованием функции-предиката |
| `apply(func)` | Применяет функцию ко всем элементам коллекции |
| `map_to(transform_func)` | Преобразует коллекцию с помощью `map()` |
| `chain()` | Начинает цепочку операций |

## 5. Вывод

В ходе выполнения лабораторной работы были изучены и применены на практике следующие концепции:

### 5.1 Передача функций как аргументов
- Функции могут быть переданы в другие функции (например, `sort_by(by_damage)`)
- Это позволяет создавать гибкие и переиспользуемые алгоритмы

### 5.2 Lambda-выражения
- Позволяют создавать анонимные функции «на лету»
- Упрощают код при простых преобразованиях: `lambda x: x._damage`

### 5.3 Функции высшего порядка
- `map()` — применяет функцию к каждому элементу коллекции
- `filter()` — отбирает элементы по условию
- `sorted()` — сортирует коллекцию с пользовательским ключом

### 5.4 Фабрики функций и замыкания
- Функции могут создавать и возвращать другие функции
- Замыкания сохраняют параметры, переданные при создании

### 5.5 Паттерн «Стратегия»
- Позволяет выносить алгоритмы в отдельные классы/функции
- Стратегии взаимозаменяемы без изменения кода коллекции
- Реализован через callable-объекты с методом `__call__()`

### 5.6 Цепочки операций
- Позволяют последовательно применять фильтрацию, сортировку и преобразования
- Делают код более читаемым и декларативным
